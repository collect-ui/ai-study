明白了,在完全无界面的 Ubuntu 命令行下开发小程序,需要一套**纯 CLI 的工具链**。好消息是,这套方案完全可行,而且自动化程度更高。

核心思路:用 **Codex CLI 写代码 + 微信开发者工具 CLI 进行预览和上传**,所有操作都在终端完成。

---

## 🎯 一、整体架构

```
Ubuntu Terminal
    ├── Codex CLI (AI 编程助手,负责写代码)
    ├── 微信开发者工具 (Linux 版,命令行调用)
    ├── leq-cli (项目脚手架,快速初始化)
    └── miniprogram-ci (CI/CD,用于上传)
```

工作流:用 Codex 写代码 → 微信开发者工具 CLI 预览 → 迭代 → miniprogram-ci 上传

---

## 🔧 二、环境搭建步骤

### 2.1 安装 Codex CLI

```bash
# 安装 Node.js (如果还没有)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 Codex CLI
npm install -g @openai/codex

# 登录 (会打开浏览器授权)
codex login

# 验证安装
codex --version
```

> **注意**:Codex CLI 首次登录需要浏览器完成 OAuth 授权。如果你的 Ubuntu 是无图形界面的服务器,可以:
> - 用 `codex login --headless` 尝试无头模式
> - 或在本地电脑登录后,将 `~/.codex/config.toml` 复制到服务器

### 2.2 安装微信开发者工具 (Linux 版)

官方没有 Linux 版,但社区有移植版:

```bash
# 下载 Linux 版微信开发者工具
wget http://oss.yuan1994.com/github/wechat_dev_tools/latest/wechat_web_devtools.tar.gz

# 解压并安装
tar -zxvf wechat_web_devtools.tar.gz
cd wechat_web_devtools
./install.sh /opt/tencent

# 添加软链接
sudo ln -s /opt/tencent/wechat_web_devtools/nw /usr/local/bin/wechat_devtools

# 启动一次,完成首次初始化(需要图形界面)
wechat_devtools
```

⚠️ **注意**:首次启动需要图形界面完成登录和设置。之后可以完全通过 CLI 操作。

**关键配置**:在开发者工具中开启"服务端口"(设置 → 安全设置 → 开启服务端口),这是 CLI 调用的前提。

### 2.3 安装 leq-cli 脚手架(可选)

用于快速生成项目模板:

```bash
npm install -g leq-cli

# 创建微信小程序项目
lq-cli create my-english-app -t 微信小程序
```

## 💻 三、开发工作流

### 3.1 用 Codex 生成小程序代码

进入项目目录,启动 Codex 交互式会话:

```bash
cd my-english-app
codex
```

在 Codex 会话中,用自然语言描述需求:

```
我需要开发一个英语学习小程序,功能包括:
1. 单词学习页面:展示单词、音标、例句,点击播放发音
2. 口语练习页面:集成语音评测,用户跟读后给出评分
3. 打卡功能:每日学习打卡,显示连续天数
4. 使用微信小程序原生开发,文件结构要符合规范
请生成完整的项目代码
```

Codex 会生成代码,然后执行:

```bash
codex apply  # 将代码应用到本地文件
```

### 3.2 实时预览和调试(CLI 方式)

微信开发者工具提供了完整的命令行接口:

```bash
# 打开项目(会自动编译刷新)
cli open --project /path/to/my-english-app

# 预览(生成二维码,用手机扫码查看)
cli preview --project /path/to/my-english-app --qr-format terminal

# 也可以将二维码输出为文件
cli preview --project /path/to/my-english-app --qr-format base64 --qr-output qr.txt
```

**预览时如果发现 Bug,回到 Codex 继续修改**:

```bash
codex
> 修复首页单词卡片的样式问题,字体太小了
codex apply
# 项目会自动刷新,再次扫码预览
```

### 3.3 上传代码

```bash
# 登录(首次需要)
cli login

# 上传代码
cli upload --project /path/to/my-english-app -v "1.0.0" -d "初版发布"
```

## 🚀 四、进阶:远程开发方案

如果你的 Ubuntu 是服务器(无图形界面),还可以更进一步:**用微信直接控制 Codex**。

安装 `weacpx`:

```bash
npm install -g weacpx

# 扫码登录微信机器人
weacpx login

# 启动后台服务
weacpx start
```

然后在微信里发指令:
```
/ss codex -d /absolute/path/to/your/repo
添加一个单词测验功能,从词库里随机出题
```

Codex 会在服务器上执行,结果通过微信返回给你。这意味着你**在手机上就能完成代码修改**。

## 📝 五、完整开发流程总结

| 步骤 | 操作 | 命令 |
|------|------|------|
| 1. 创建项目 | 脚手架初始化 | `lq-cli create my-app -t 微信小程序` |
| 2. 生成代码 | Codex 写代码 | `codex` → 描述需求 → `codex apply` |
| 3. 预览 | CLI 生成二维码 | `cli preview --project ./my-app --qr-format terminal` |
| 4. 调试迭代 | 发现问题 → Codex 修改 → 重新预览 | `codex exec "修复xxx问题"` → `codex apply` |
| 5. 上传 | CLI 上传代码 | `cli upload --project ./my-app -v "1.0.0"` |

---

## ⚠️ 六、注意事项

1. **首次启动需要图形界面**:微信开发者工具 Linux 版首次运行需要图形界面完成登录授权。可以在本地电脑完成后再部署到服务器。

2. **开启服务端口**:在开发者工具设置中开启"服务端口"(默认端口约 10000+),否则 CLI 无法调用。

3. **项目配置文件**:确保项目根目录有正确的 `project.config.json`,包含 appid 和 projectname。

4. **代码安全**:Codex 生成代码后,建议用 `git` 管理,每次 `codex apply` 前先提交,方便回滚。

---

这套方案让你在纯终端环境下,通过自然语言驱动开发,从零到上线完全不需要打开任何图形界面。你的 Ubuntu 服务器可以放在任何地方,你只需要 SSH 进去,然后让 Codex 帮你写代码。
