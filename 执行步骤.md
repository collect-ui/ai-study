# 微信小程序本机开发执行步骤

本文档按当前机器 `/data/project/ai-study` 已安装好的环境编写。照着操作即可开始写微信小程序。

## 1. 已安装工具

当前已准备好这些命令：

```bash
node -v
npm -v
codex --version
lq-cli -v
miniprogram-ci --version
cli --help
```

其中：

- `codex`：写代码用。
- `lq-cli`：创建微信小程序项目模板用。
- `cli`：微信开发者工具 CLI，用于打开、预览、上传。
- `miniprogram-ci`：CI 上传工具，需要微信小程序后台的上传密钥。

微信开发者工具安装在：

```bash
/data/project/ai-study/.tools/WeChat_Dev_Tools_v2.01.2510290-2_x86_64_linux
```

命令入口已经挂到 PATH：

```bash
wechat-devtools
wechat_devtools
wechat-devtools-cli
cli
```

## 2. 首次登录微信开发者工具

微信开发者工具首次使用必须扫码登录一次。当前机器是无图形 TTY 环境，不能直接显示登录二维码窗口。

如果你有图形桌面，直接执行：

```bash
wechat-devtools
```

登录后，在工具里打开：

```text
设置 -> 安全设置 -> 服务端口 -> 开启
```

如果你通过 SSH 使用这台机器，推荐任选一种方式完成首次登录：

```bash
# 方案 A：使用 X11 转发登录
ssh -X zz@你的机器地址
cd /data/project/ai-study
wechat-devtools
```

```bash
# 方案 B：在另一台有桌面的 Linux 机器登录后，复制配置目录
rsync -av ~/.config/wechat-devtools/ zz@你的机器地址:~/.config/wechat-devtools/
```

说明：`xvfb-run` 可以让工具在无显示环境启动，但你看不到登录窗口，所以它不适合首次扫码登录。

## 3. 创建微信小程序项目

进入工作目录：

```bash
cd /data/project/ai-study
```

创建项目：

```bash
lq-cli create my-mini-app -t 微信小程序
```

进入项目：

```bash
cd my-mini-app
```

确认项目里有小程序配置文件：

```bash
ls project.config.json app.json
```

如果 `project.config.json` 里的 `appid` 还是测试值，改成你的小程序 AppID。

## 4. 用 Codex 写代码

在项目目录执行：

```bash
codex
```

示例需求：

```text
我要开发一个英语学习微信小程序。
需要首页、单词学习页、打卡页。
使用微信小程序原生开发，生成完整页面、样式和示例数据。
```

如果只想让 Codex 执行一次修改，也可以用：

```bash
codex exec "给这个微信小程序增加一个单词测验页面"
```

## 5. 打开项目

首次登录和服务端口开启后，在项目目录执行：

```bash
cli open --project "$(pwd)"
```

如果当前是无图形环境，但已经完成登录，可以用虚拟显示启动：

```bash
xvfb-run -a cli open --project "$(pwd)" --disable-gpu
```

## 6. 手机预览

在项目目录执行：

```bash
cli preview --project "$(pwd)" --qr-format terminal
```

终端会显示二维码，用微信扫码预览。

如果要把二维码保存到文件：

```bash
cli preview --project "$(pwd)" --qr-format base64 --qr-output qr.txt
```

## 7. 构建 npm

如果项目使用 npm 依赖，先执行：

```bash
npm install
cli build-npm --project "$(pwd)"
```

## 8. 上传版本

使用微信开发者工具 CLI 上传：

```bash
cli upload --project "$(pwd)" -v "1.0.0" -d "初版"
```

也可以使用 `miniprogram-ci`，但需要先在微信公众平台下载上传密钥，并在项目中配置上传脚本。

## 9. 常用检查命令

检查微信开发者工具 CLI：

```bash
cli --help
```

检查是否已经登录：

```bash
cli islogin
```

如果无图形环境：

```bash
xvfb-run -a cli islogin --disable-gpu
```

关闭开发者工具：

```bash
cli quit
```

## 10. 常见问题

### 提示服务端口关闭

先打开微信开发者工具，进入：

```text
设置 -> 安全设置 -> 服务端口 -> 开启
```

然后重试：

```bash
cli islogin
```

### 无图形环境无法扫码登录

必须先用有图形的方式完成一次登录。可以使用 X11 转发、远程桌面，或从已登录机器复制：

```bash
~/.config/wechat-devtools/
```

### `wechat-devtools` 缺 GTK 崩溃

当前项目已内置本地 GTK 运行库，并通过包装脚本自动注入环境变量。正常使用 PATH 里的命令即可：

```bash
wechat-devtools
cli --help
```

不要直接运行 `.tools/WeChat_Dev_Tools.../bin/` 里的原始脚本。

### npm 下载失败

当前 npm 默认源是内网源。如果安装依赖失败，可以临时改用公网源：

```bash
npm install --registry=https://registry.npmjs.org/
```
